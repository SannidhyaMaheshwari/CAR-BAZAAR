==================================================
3. BACKEND FILE-BY-FILE EXPLANATION
==================================================

main.py
- Entry point of FastAPI application
- Registers all routers
- Starts CAR-BAZAAR backend server

core/database.py
- Creates MongoDB client using connection string
- Exposes database object for reuse across app

core/security.py
- Handles password hashing and verification
- Uses Argon2 for secure password storage

users/models.py
- Defines Pydantic schemas for users
- Validates input data during registration
- Ensures passwords are never returned in responses

auth/routes.py
- Defines authentication API endpoints
- Handles HTTP requests for register and login
- Calls service layer for business logic

auth/services.py
- Contains core authentication logic
- Hashes passwords
- Verifies login credentials
- Interacts directly with MongoDB collections

==================================================
6. AUTHENTICATION FLOW (CURRENT)
==================================================

1. User registers with:
   - name
   - email
   - password
   - role (buyer / seller)

2. Password is hashed using Argon2

3. User data stored in MongoDB (users collection)

4. User logs in using email and password

5. Credentials are verified securely

6. User details and role are returned
